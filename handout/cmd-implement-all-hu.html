/implement-all

← Vissza az áttekintéshez

Összefoglaló

Egy /goal ciklust futtat, amely egyszerre egy feladatot delegál a /implement-next parancsnak — addig, amíg a tervfájl minden befejezetlen feladata implementálva és commitolva nem lesz.

Mikor használd

Akkor használd a /implement-all parancsot, ha van egy tervfájlod több befejezetlen feladattal, és a teljes implementációs ciklust kézi beavatkozás nélkül szeretnéd lefuttatni. A parancs addig léptet, amíg a terv el nem készül — feladatonként egy-egy ügynököt indít, amelyek mindegyike a /implement-next parancsra delegál.

  • Van egy tervfájlod (pl. plans/FEAT-001.md) egy vagy több [ ] feladattal, és sorban mindet implementálni szeretnéd.
  • Azt akarod, hogy minden feladat saját, nem üres fájlváltozásokat tartalmazó commitot kapjon (ezt a /implement-next kényszeríti ki).
  • Egy részben befejezett futást folytatsz, és a következő befejezetlen feladatnál akarsz továbblépni.

Kritikus megszorítások (a parancs maga kényszeríti ki):

  • NEM indít egyetlen ügynököt az összes feladatra — minden feladat külön-külön megy át a /implement-next parancson.
  • NEM ír közvetlenül kódot — minden implementáció delegálva van.
  • NEM lép tovább, ha a terv feladatszakaszának fejléce hiányzik (WARNING-ot dob és megáll).

Hívás

Egyetlen argumentumként add át a tervfájl elérési útját.

/implement-all [plan-file-path]

Paraméterek:

  • [plan-file-path] — (kötelező) A terv Markdown-fájljának relatív elérési útja, pl. plans/FEAT-001-my-feature.md.

Folyamat (/goal ciklus):

  1. Haladás ellenőrzése: Lefuttatja a bash ~/.claude/scripts/plan-progress.sh <plan-file> parancsot.
    • Exit 1 → minden feladat kész — a cél teljesült, megáll.
    • Exit 2 vagy 3 → megáll és jelenti a hibát.
    • Exit 0 → vannak még feladatok, folytatódik a 2. lépéssel.
  2. Delegálás: Indít egy ügynököt, amely a /implement-next parancsot hívja a tervfájlon.
  3. Ciklus: Visszatér az 1. lépésre, amíg minden feladat el nem készül.

1. példa — Egy feature-terv összes feladatának futtatása (szemléltető példa)

Bemenet:

/implement-all plans/FEAT-001-skill-command-script-handouts.md

Kimenet / Eredmény:

Goal: All tasks in plans/FEAT-001-skill-command-script-handouts.md are marked complete.

[1. iteráció] plan-progress.sh → 0/7 kész, következő: Task 3.3 — cmd-implement-all.html
  → /implement-next ügynök indítása… kész (commit b1d2e3f)

[2. iteráció] plan-progress.sh → 1/7 kész, következő: Task 3.4 — cmd-implement-next.html
  → /implement-next ügynök indítása… kész (commit c2e3f40)

… (5 további iteráció) …

[8. iteráció] plan-progress.sh → Minden feladat kész. (exit 1)
Cél teljesítve.

2. példa — Folytatás részleges futás után (szemléltető példa)

Bemenet:

/implement-all plans/FEAT-002-api-refactor.md

Kimenet / Eredmény:

Goal: All tasks in plans/FEAT-002-api-refactor.md are marked complete.
(4 feladat már [x]-szel jelölve egy korábbi futásból)

[1. iteráció] plan-progress.sh → 4/7 kész, következő: Task 2.3 — Extract AuthService
  → /implement-next ügynök indítása… kész (commit d4e5f60)

[2. iteráció] plan-progress.sh → 5/7 kész, következő: Task 2.4 — Wire new AuthService
  → /implement-next ügynök indítása… kész (commit e5f6071)

[3. iteráció] plan-progress.sh → 6/7 kész, következő: Task 2.5 — Update integration tests
  → /implement-next ügynök indítása… kész (commit f607182)

[4. iteráció] plan-progress.sh → Minden feladat kész. (exit 1)
Cél teljesítve.

Kapcsolódó eszközök

  • /implement-next — Az egyfeladatos implementációs parancs, amelynek a /implement-all minden feladatot átad; egyszerre egy feladatra futtatja a TDD-t, a DA review-t, a teszteket, a kipipálást és a commitot.
  • /iterative-review — Devil's advocate review-ciklus, amelyet a /implement-next hív meg minden feladat implementációja során; párhuzamos DA ügynököket és fix ügynököket indít, amíg nem marad critical/major/moderate probléma.
