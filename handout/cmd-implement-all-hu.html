/implement-all (portable változat)

← Vissza az áttekintéshez

Összefoglaló

Runtime-független portable ciklus, amely egyszerre egy feladatot delegál a /implement-next parancsnak — addig, amíg a tervfájl minden befejezetlen feladata implementálva és commitolva nem lesz. Bármilyen harness-ben (Claude Code, Cursor, claude -p) működik, hook-függőség nélkül.

Két változat létezik. Ez az oldal a portable változatot írja le. A Claude Code hook-kényszerített, auto-rescue képes változatáért (Case A/B/C/D recovery + SubagentStop gate) lásd a /implement-all-cc parancsot.

Használat Cursorban (egyszeri beállítás)

A Cursor a projekten belüli .cursor/commands/ könyvtárból olvassa a custom parancsokat, nem a ~/.claude/commands/-ból. Ahhoz, hogy ezt a skill-t Cursorban használd, másold vagy szimlinkeld a portable command-fájlokat:

mkdir -p .cursor/commands
ln -s ~/.claude/commands/implement-all.md .cursor/commands/implement-all.md
ln -s ~/.claude/commands/implement-next.md .cursor/commands/implement-next.md

A Cursor Task tool primitívje a Claude Code Agent tooljával egyenértékűen működik. A portable változat NEM használ hook-okat — nincs további teendő.

Mikor használd

Akkor használd a /implement-all parancsot, ha van egy tervfájlod több befejezetlen feladattal, és a teljes implementációs ciklust kézi beavatkozás nélkül szeretnéd lefuttatni. A parancs addig léptet, amíg a terv el nem készül — feladatonként egy-egy ügynököt indít, amelyek mindegyike a /implement-next parancsra delegál.

  • Van egy tervfájlod (pl. plans/FEAT-001.md) egy vagy több [ ] feladattal, és sorban mindet implementálni szeretnéd.
  • Azt akarod, hogy minden feladat saját, nem üres fájlváltozásokat tartalmazó commitot kapjon (ezt a /implement-next kényszeríti ki).
  • Egy részben befejezett futást folytatsz, és a következő befejezetlen feladatnál akarsz továbblépni.

Kritikus megszorítások (a parancs maga kényszeríti ki):

  • NEM indít egyetlen ügynököt az összes feladatra — minden feladat külön-külön megy át a /implement-next parancson.
  • NEM ír közvetlenül kódot — minden implementáció delegálva van.
  • NEM lép tovább, ha a terv feladatszakaszának fejléce hiányzik (WARNING-ot dob és megáll).

Hívás

Egyetlen argumentumként add át a tervfájl elérési útját.

/implement-all [plan-file-path]

Paraméterek:

  • [plan-file-path] — (kötelező) A terv Markdown-fájljának relatív elérési útja, pl. plans/FEAT-001-my-feature.md.

Folyamat (0. lépés + 5 ciklus-lépés):

  1. 0. lépés — Tervfájl feloldása: Ha az argumentum egy létező fájl elérési útja, azt használja. Egyébként a **/*.md mintára illesztett kulcsszó-egyezést keres olyan jelöltek között, amelyek tartalmaznak legalább egy be nem pipált taskot. Megáll, ha egyértelműség nélkül több vagy egy találat sincs.

Ciklustörzs (iterációs felső korlát: 50; megakadt-feladat őr: ugyanaz a NEXT_TASK_NAME két iterációban → megáll):

  1. Haladás + megakadt-feladat őr: Lefuttatja a bash ~/.claude/scripts/plan-progress.sh <plan-file> parancsot. Exit 1 → minden kész, megáll. Exit 2/3 → hibajelentéssel megáll. Exit 0 → megjegyzi a NEXT_TASK_NAME-et és folytatódik.
  2. Pre-state mentése: rögzíti a START_SHA és CWD értékeket. Csak az 1. iterációban a start SHA-t kiírja a $CWD/.claude/implement-all-start-sha-<8-hex-hash-of-plan-path> fájlba is, hogy az 5. lépés auditja ne függjön egy ideiglenes változótól. Minden tervhez saját SHA-horgonyfájl tartozik (a terv elérési útjának hash-éval névterezve), így különböző tervek nem ütköznek halt-and-resume esetén.
  3. Subagent indítása: a runtime saját subagent-primitívjével (Claude Code Agent tool, Cursor Task tool, vagy headless harness-ben inline /implement-next) ezzel a prompttal: "Run /implement-next on plan file <plan-path>." Megvárja a visszatérést.
  4. Recovery check — landolt-e a feladat: a hook-kényszerítés portable helyettesítője. Bármely hibamódra megáll (nem auto-rescue):
    • END_SHA != START_SHA & tiszta munkamásolat → valószínűleg commitolva; újra plan-progress.sh, és folytatódik (vagy megáll, ha a task nincs kipipálva).
    • END_SHA == START_SHA & commitolatlan változások → a subagent megakadt menet közben (a domináns hibatípus); diagnosztikával megáll. Javasolja a Claude Code /implement-all-cc változatát.
    • END_SHA == START_SHA & tiszta munkamásolat → a subagent semmit sem csinált láthatóan (hálózat, OOM, kvóta). Megáll.
    • END_SHA != START_SHA & commitolatlan változások → anomália; megáll.
  5. Audit: a ciklus rendes lezárása után lefuttatja a bash ~/.claude/scripts/audit-plan-run.sh <plan-path> <FIRST_START_SHA> parancsot, majd takarítja a $CWD/.claude/implement-all-start-sha-<8-hex-hash-of-plan-path> fájlt. Exit 0 = tiszta futás; exit 1 = eltérés.

Miért kevésbé megbízható ez, mint a -cc változat: arra támaszkodik, hogy a subagent kövesse az /implement-next markdown utasításait a commit elvégzéséhez a turn vége előtt. Az archon-search projekt korábbi tranzakcióiban a /implement-next subagentek ~7,6%-a soha nem commitolt (ez a fő hibamód, amelyre ez a skill reagál). A Plan-and-Solve tanulmány (arxiv.org/abs/2305.04091, hozzáférve: 2026-06-11) minőségileg nevezi meg a "missing-step" hibákat mint ismert hibakategóriát prompt-vezérelt többlépéses flow-knál — a mi mérésünk ennek egy konkrét megnyilvánulása. A megállás korlátozza a kárt, de nem auto-recovery.

Kapcsolódó eszközök

  • /implement-all-cc — Claude Code változat SubagentStop hook-kényszerítéssel és Case A/B/C/D parent-szintű recovery-vel; használd Claude Code-ban a nagyobb megbízhatóságért.
  • /implement-next — A portable egyfeladatos implementációs parancs, amelynek az /implement-all minden feladatot átad; egyszerre egy feladatra futtatja a TDD-t, az opcionális kritikai review-t, a teszteket, a kipipálást, a commitot és a self-verification ellenőrzést.
  • /implement-next-cc — Az /implement-next Claude Code változata; az /implement-all-cc párja, a SubagentStop hook gátolja a commit nélküli turn-end-et.
  • /iterative-review — Devil's advocate review-ciklus, amelyet az /implement-next hív meg minden feladat implementációja során.
  • scripts-plan — A plan-progress.sh és audit-plan-run.sh szkriptek, amelyeket ez a parancs hív.
