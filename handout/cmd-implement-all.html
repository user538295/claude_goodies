/implement-all

← Back to index

Summary

Repeatedly delegates to /implement-next — one task at a time — until every uncompleted task in a plan file is implemented, verified, and committed.

When to use

Use /implement-all when you have a plan file with multiple uncompleted tasks and want to run the full implementation loop hands-off. The command handles the entire cycle — building a task list, spawning implementation agents, verifying each commit, running a final audit, and fixing any test failures — without requiring manual intervention between tasks.

  • You have a plan file (e.g. plans/FEAT-001.md) with one or more [ ] tasks and want to implement them all in sequence.
  • You want each task to get its own commit with non-empty file changes, enforced automatically.
  • You want a final audit (verify-run-commits.sh) to confirm commit granularity after all tasks complete.
  • You are resuming a partially-completed run and want to continue from the next uncompleted task.

Critical constraints (enforced by the command itself):

  • Does NOT spawn a single agent for all tasks — every task goes through /implement-next individually.
  • Does NOT write code directly — all implementation is delegated.
  • Does NOT skip or merge verify tasks.
  • Does NOT proceed if the plan's ## Tasks or ## Task breakdown heading is missing (raises a WARNING and stops).

Invocation

Pass the path to the plan file as the sole argument.

/implement-all [plan-file-path]

Parameters:

  • [plan-file-path] — (required) Relative path to the plan Markdown file, e.g. plans/FEAT-001-my-feature.md.

Process phases:

  1. Setup: Records sha_start (current HEAD) and runs count-uncompleted-tasks.sh to get the exact task count. Stops immediately if a WARNING is raised.
  2. Build task list: Creates 2 × N tasks — alternating [IMPLEMENT] / [VERIFY] pairs, one pair per uncompleted plan task.
  3. Execute pairs: For each pair, spawns an implementation agent (via /implement-next), then a verification agent that checks: new commit exists, check-task-commit.sh passes, plan item is marked [x], and checkpoint tests pass.
  4. Final audit: Runs verify-run-commits.sh <sha_start> <N> and the full test suite. Spawns fix agents for any failing tests (up to 3 attempts).

Zero tasks: If count-uncompleted-tasks.sh returns 0 (and no WARNING), the task-list build is skipped and the command jumps directly to the final audit.

Resuming a partial run: Re-record sha_start at the current HEAD and re-run — count-uncompleted-tasks.sh counts only the remaining [ ] tasks. To audit the full run from the beginning, use bash ~/.claude/scripts/audit-plan-run.sh <plan-file> <original_sha_start>.

Example 1 — Run all tasks in a feature plan (illustrative example)

Input:

/implement-all plans/FEAT-001-skill-command-script-handouts.md

Output / Result:

sha_start = a3f9c12

count-uncompleted-tasks.sh output: number_of_uncompleted_tasks=7

Building task list (14 tasks):
  [IMPLEMENT] Task 3.3 — cmd-implement-all.html
  [VERIFY]    Task 3.3 — cmd-implement-all.html
  [IMPLEMENT] Task 3.4 — cmd-implement-next.html
  [VERIFY]    Task 3.4 — cmd-implement-next.html
  …

[IMPLEMENT Task 3.3] → spawning /implement-next agent…
[VERIFY Task 3.3]    → new commit b1d2e3f detected; check-task-commit.sh: PASS; plan [x]: PASS → PASS

… (6 more pairs) …

Final audit: verify-run-commits.sh a3f9c12 7 → EXIT 0 (PASS)
Full test suite: all 42 tests green (CLEAN)

Summary: 7 tasks implemented, 7 commits, audit PASS, suite CLEAN.

Input:

/implement-all plans/FEAT-002-api-refactor.md

Related tools

  • /implement-next — The single-task implementation command that /implement-all delegates to for each task; runs TDD, DA review, tests, check-off, and commit for one task at a time.
  • /iterative-review — Devil's advocate review loop invoked by /implement-next during each task's implementation; spawns parallel DA agents and fix agents until no critical/major/moderate issues remain.
