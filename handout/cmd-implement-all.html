/implement-all

← Back to index

Summary

Portable runtime-agnostic loop that delegates to /implement-next — one task at a time — until every uncompleted task in a plan file is implemented and committed. Works in any harness (Claude Code, Cursor, claude -p) without hook dependencies.

Using in Cursor (one-time setup)

Cursor reads custom commands from .cursor/commands/ in your project, not from ~/.claude/commands/. To use this skill in Cursor, copy or symlink the portable command files:

mkdir -p .cursor/commands
ln -s ~/.claude/commands/implement-all.md .cursor/commands/implement-all.md
ln -s ~/.claude/commands/implement-next.md .cursor/commands/implement-next.md

Cursor's Task tool primitive translates equivalently to Claude Code's Agent. The portable variant does NOT use hooks — no further setup.

When to use

Use /implement-all when you have a plan file with multiple uncompleted tasks and want to run the full implementation loop hands-off. The command loops until the plan is complete — spawning one agent per task, each delegating to /implement-next — without requiring manual intervention between tasks.

  • You have a plan file (e.g. plans/FEAT-001.md) with one or more [ ] tasks and want to implement them all in sequence.
  • You want each task to get its own commit with non-empty file changes (enforced by /implement-next).
  • You are resuming a partially-completed run and want to continue from the next uncompleted task.

Critical constraints (enforced by the command itself):

  • Does NOT spawn a single agent for all tasks — every task goes through /implement-next individually.
  • Does NOT write code directly — all implementation is delegated.
  • Does NOT proceed if the plan's task section heading is missing (raises a WARNING and stops).

Invocation

Pass the path to the plan file as the sole argument.

/implement-all [plan-file-path]

Parameters:

  • [plan-file-path] — (required) Relative path to the plan Markdown file, e.g. plans/FEAT-001-my-feature.md.

Process (Step 0 + loop):

  1. Step 0 — Resolve the plan file: If the argument is a path to an existing file, use it. Otherwise, glob **/*.md and keyword-match candidates with at least one unchecked task. Halt on ambiguity or no match.

Loop body (repeats until all tasks complete):

  1. Progress: Runs bash ~/.claude/scripts/plan-progress.sh <plan-file> and shows the first two lines of output. Exit 1 → all tasks complete, stop. Exit 2/3 → halt with error. Exit 0 → note NEXT_TASK_NAME and continue.
  2. Spawn subagent: Uses the runtime's subagent primitive (Claude Code Agent tool, Cursor Task tool, or inline /implement-next in headless harnesses) with a strict scoped prompt. The prompt enforces ONE-task-only, requires Steps A–F completion (implementation, /iterative-review, tests pass, plan checked off, single commit, report), and forbids --no-verify / --amend / bundling tasks / nested /implement-all calls / scope creep. Then waits for the subagent to return.
  3. Recovery check — verify the task landed: Checks that the task is checked off in the plan file and that related files are committed. If either is missing, re-spawns the subagent for this task (Step 2) and repeats. Reports any anomalies to the user.

Example 1 — Run all tasks in a feature plan (illustrative example)

Input:

/implement-all plans/FEAT-001-skill-command-script-handouts.md

Output / Result:

[Iteration 1] plan-progress.sh → 0/7 complete, next: Task 3.3 — cmd-implement-all.html
  → spawning /implement-next agent… done (commit b1d2e3f)

[Iteration 2] plan-progress.sh → 1/7 complete, next: Task 3.4 — cmd-implement-next.html
  → spawning /implement-next agent… done (commit c2e3f40)

… (5 more iterations) …

[Iteration 8] plan-progress.sh → All tasks complete. (exit 1)
All tasks complete.

Example 2 — Resume after a partial run (illustrative example)

Input:

/implement-all plans/FEAT-002-api-refactor.md

Output / Result:

(4 tasks already marked [x] from a prior session)

[Iteration 1] plan-progress.sh → 4/7 complete, next: Task 2.3 — Extract AuthService
  → spawning /implement-next agent… done (commit d4e5f60)

[Iteration 2] plan-progress.sh → 5/7 complete, next: Task 2.4 — Wire new AuthService
  → spawning /implement-next agent… done (commit e5f6071)

[Iteration 3] plan-progress.sh → 6/7 complete, next: Task 2.5 — Update integration tests
  → spawning /implement-next agent… done (commit f607182)

[Iteration 4] plan-progress.sh → All tasks complete. (exit 1)
All tasks complete.

Related tools

  • /implement-next — The single-task implementation command that /implement-all delegates to for each task; runs TDD, critical review, tests, check-off, and commit.
  • /iterative-review — Devil's advocate review loop invoked by /implement-next during each task's implementation; spawns parallel DA agents and fix agents until no critical/major/moderate issues remain.
  • scripts-plan — The plan-progress.sh script called by this command.
