/implement-next-cc (Claude Code változat)

← Vissza az áttekintéshez

Összefoglaló

Az /implement-next Claude Code változata: egy feladatot valósít meg egy tervfájlból TDD-vel, párhuzamos DA review-kkal, blokkoló tesztekkel, kipipálással és committal. Az /implement-all-cc párja, amelynek SubagentStop hookja gátolja a turn lezárását, amíg egy commit le nem landol. A szülő-szintű Case B rescue-t egy dedikált skill, az /implement-next-cc-resume kezeli — azt soha ne hívd közvetlenül.

Két változat létezik. Ez az oldal a Claude Code változatot írja le. Egy hook-függetlenül, runtime-függetlenül működő portable változatért lásd a /implement-next parancsot.

Mikor használd

Akkor használd az /implement-next-cc parancsot, ha Claude Code-ban futsz, és egy feladatot szeretnél implementálni teljes minőségi szűrőkkel (TDD, DA review, sikeres tesztsuite, commit), SubagentStop hook védelemmel, amely gátolja a commit nélküli turn-end-et. Ez az az atomi egység, amelyet az /implement-all-cc hív.

  • Van egy tervfájlod, és csak a következő taskot akarod megvalósítani Claude Code-ban.
  • Hook-kényszerített commit-előtt-turn-end viselkedést akarsz.
  • Kézzel haladsz végig egy terven Claude Code-on belül, feladatonként.
  • Egy elhasalt /implement-all-cc futást debugolsz, és izoláltan újra szeretnél futtatni egy taskot.

Kapcsolat az /implement-all-cc paranccsal: az /implement-all-cc minden el nem végzett feladatra indít egy /implement-next-cc háttér-agentet, és felfegyverzi a SubagentStop hookot egy agentId-re szűrt sentinellel. A hook gátolja a subagent turn lezárását, amíg új commit nem létezik a START_SHA óta. A hookot megkerülő hibákat (OOM, transport, kvóta) a szülő Case A/B/C/D recovery-je kapja el.

Rescue mód: amikor az /implement-all-cc megakadt subagentet detektál (Case B), meghívja a dedikált /implement-next-cc-resume skill-t, amely csak kipipálást + committot hajt végre a már stage-elt munkán. A rescue skill külön parancs — ez a skill már nem flag-vezérelt.

Hívás

Egyetlen argumentumként add át a tervfájl elérési útját.

/implement-next-cc [plan-file-path]

Paraméterek:

  • [plan-file-path] — (kötelező) A terv Markdown-fájljának elérési útja, pl. plans/FEAT-001-my-feature.md.

Megjegyzés a rescue-ról: Case B rescue esetén (a szülő ellenőrizte az implementációt + a teszteket, de nem landolt commit) a szülő egy külön skill-t, az /implement-next-cc-resume-t használja, amely csak kipipálást + committot hajt végre a már stage-elt munkán. Ne adj át "resume" flag-et ennek a parancsnak — az a flag már nem létezik.

Minden feladatra végrehajtott lépések (7 lépés összesen):

  1. 1. lépés — Haladás kijelzése: Lefuttatja a plan-progress.sh-t a jelenlegi készültség kijelzéséhez és a következő feladat azonosításához.
  2. 2. lépés — Implementáció (TDD): A legalkalmasabb ágenst indítja, hogy kizárólag ezt a feladatot valósítsa meg. Kódfeladatoknál: előbb bukó tesztek (red), aztán implementáció (green), végül teljes pass. Nem kódfeladatoknál: közvetlen megvalósítás.
  3. 3. lépés — DA review-hurok: Meghívja az /iterative-review-t a git diff HEAD és a feladat specifikációjával. Tartalmazza a hangos "KÖTELEZŐ FOLYTATÁS — ne állj meg itt" figyelmeztetést a review verdikt utáni turn-end ellen. (A SubagentStop hook a másodlagos védelem — lásd a 4. lépést.)
  4. 4. lépés — Tesztek futtatása: Subagent-specifikus útmutatás — figyelmesen olvasd:
    • A tesztek 9 perc alatt befejeződnek (preferált): teljes suite Bash foreground-dal, timeout: 540000. Blokkol, amíg visszatér.
    • A tesztek NEM férnek bele 9 percbe: csak a feladat-releváns részhalmazt futtasd, és zárd a turnt commitolatlan munkamásolattal, de a releváns részhalmaz zölden. Az /implement-all-cc Case B recovery-je szülő szinten futtatja a teljes suite-ot (ahol a Monitor működik).
    Soha ne használd a Monitor tool-t ebben a subagent-ben tesztek várására — a Monitor arra utasítja az agentet, hogy yield-eljen, de a subagent-nek nincs jövőbeli user turn, amihez az events kapcsolódhatnának. A futás csendben elhal, és a szülő üres committal találkozik. Ez volt a történelmi implement-next futások domináns hibatípusa.
  5. 5. lépés — Kipipálás: Egy ágenst indít, hogy [x]-szel jelölje a feladatot és minden elkészült alpontot a tervfájlban.
  6. 6. lépés — Commit: Stage-eli az implementációs fájlokat és a frissített tervfájlt; a commit üzenete a feladat tartalmából származik. A --no-verify tilos. Az /implement-all-cc-ből hívva a SubagentStop hook olvassa a $CWD/.claude/implement-next-state.json sentinelt és nem engedi lezárni a turnt, amíg új commit nem létezik a START_SHA óta.
  7. 7. lépés — Riport: Tömör összefoglaló: mit valósított meg, mik a teszteredmények, milyen eltérések voltak, mit pipált ki.

1. példa — Hook-gátolt egyfeladatos implementáció (szemléltető példa)

Bemenet (ahogyan az /implement-all-cc hívja):

Call the Skill tool with skill `implement-next-cc` and args `plans/FEAT-001-skill-command-script-handouts.md`

Kimenet / Eredmény:

Progress   : [████████░░░░] 68%  (13/19 tasks, 2/6 phases)
Next task  : 3.4  **File**: `handout/cmd-implement-next-cc.html`

NEXT_TASK_NAME=**File**: `handout/cmd-implement-next-cc.html`
NEXT_TASK_PHASE=3
NEXT_TASK_NUMBER=4

[Step 2] Implementációs agent indítása Task 3.4-re…
  → Olvassa: commands/implement-next-cc.md
  → Írja: handout/cmd-implement-next-cc.html

[Step 3] /iterative-review meghívása…
  → DA agent: nincs kritikus/súlyos/közepes találat (PASS)

[Step 4] Teljes tesztsuite (timeout 540000 ms): minden strukturális ellenőrzés átment (CLEAN)

[Step 5] Plan frissítve: Task 3.4 [x]-szel jelölve

[Step 6] Commit: feat(handout): Task 3.4 — cmd-implement-next-cc.html
  → SubagentStop hook leellenőrizte az új commitot a START_SHA óta → turn-end megengedve

[Step 7] Riport: Task 3.4 kész.

2. példa — Case B rescue külön /implement-next-cc-resume skill-lel (szemléltető példa)

Bemenet (ahogyan az /implement-all-cc Case B rescue-ja hívja — nem ezt a skill-t, hanem a rescue skill-t):

Call the Skill tool with skill `implement-next-cc-resume` and args `plans/FEAT-002-api-refactor.md`

Kimenet / Eredmény:

[Rescue skill: a szülő ellenőrizte az implementációt + a teszteket — csak kipipálás + commit]

[1. lépés] Plan frissítve: Task 2.4 [x]-szel jelölve

[2. lépés] Commit: feat(auth): Task 2.4 — Wire new AuthService
  → SubagentStop hook leellenőrizte az új commitot a START_SHA óta → turn-end megengedve

[3. lépés] Riport: Rescue commit landolt: e5f6071

A rescue skill teljes dokumentációja: /implement-next-cc-resume.

Kapcsolódó eszközök

  • /implement-next-cc-resume — Dedikált rescue skill, amelyet az /implement-all-cc Case B útja hív; csak kipipálást + committot végez a már stage-elt munkán. Ennek a skill-nek a régi flag-vezérelt rescue módját váltja le.
  • /implement-next — Portable runtime-független párja hook-függőség nélkül; Claude Code-on kívül ezt használd.
  • /implement-all-cc — A Claude Code szülő ciklus, amely erre a parancsra delegál és felfegyverzi a SubagentStop gate-et.
  • /implement-all — Portable szülő-ciklus pár.
  • /iterative-review — A 3. lépésben meghívott DA review-ciklus.
  • scripts-plan — Tartalmazza a plan-progress.sh (1. lépés) szkriptet, plusz a hook + sentinel szkripteket implement-next-stop-gate.sh, implement-next-state-write.sh, implement-next-state-clear.sh.
