scripts-logging

← Vissza az áttekintéshez

Összefoglaló

Három shell szkript (prompt_log_lib.sh, prompt_log_new_session.sh, prompt_log_save.sh), amelyek automatikusan rögzítenek minden, a Claude Code-nak elküldött promptot projektenkénti Markdown naplófájlokba, sessionönként rendezve.

Mikor használd

Ezek a szkriptek automatikusan futnak — normál használat során nem kell kézzel meghívni őket. A settings.json-ban Claude Code hookokhoz vannak bekötve:

  • prompt_log_new_session.sh — a SessionStart hook hívja minden új Claude Code session indulásakor
  • prompt_log_save.sh — a UserPromptSubmit hook hívja minden alkalommal, amikor promptot küldesz
  • prompt_log_lib.sh — belső, megosztott függvénykönyvtár; a fenti két szkript source-olja, közvetlenül soha nem hívod

Kézzel csak akkor indítsd őket, ha a logging beállítást debugolod, vagy ha olyan szkriptet írsz, amelynek programozottan kell létrehoznia vagy bővítenie egy session naplófájlt.

Meghívás

prompt_log_new_session.sh — Session inicializáló

A SessionStart hook automatikusan hívja. JSON payloadot olvas a stdin-ről.

echo '{"session_id":"abc123","cwd":"/path/to/project"}' | bash prompt_log_new_session.sh

Új, időbélyeggel ellátott session naplófájlt hoz létre a ~/.claude/prompt-logs/<project-slug>/session_YYYY-MM-DD_HH-MM-SS.md alatt, és session map bejegyzést ír a ~/.claude/session-maps/<session_id> útvonalra.

prompt_log_save.sh — Prompt hozzáfűző

A UserPromptSubmit hook automatikusan hívja. JSON payloadot olvas a stdin-ről.

echo '{"session_id":"abc123","prompt":"your prompt text","cwd":"/path/to/project"}' | bash prompt_log_save.sh

A prompt szövegét egy HH:MM:SS időbélyeges címsorral fűzi hozzá a session naplófájlhoz. Ha még nincs session map, belsőleg meghívja a create_session_file függvényt (elegáns késői inicializálás).

prompt_log_lib.sh — Megosztott függvénykönyvtár (belső)

Közvetlenül nem hívod. A create_session_file(session_id, cwd) függvényt biztosítja. A fenti két szkript tetején source-olva van.

source "$HOME/.claude/scripts/prompt_log_lib.sh"

1. példa — Új session inicializálása

Trigger: egy új Claude Code session indul egy projektkönyvtárban (automatikus a SessionStart hookon keresztül).

Tényleges bemenet (a hook által stdin-re küldött JSON payload):

{
  "session_id": "a1b2c3d4e5f6",
  "cwd": "/Users/alice/projects/my-app"
}

Eredmény:

# Prompts — 2026-06-05 09:14:32

**Session ID:** a1b2c3d4e5f6
**Resume:** `claude --resume a1b2c3d4e5f6`
**Session file:** `~/.claude/projects/Users-alice-projects-my-app/a1b2c3d4e5f6.jsonl`

---

A fájl a ~/.claude/prompt-logs/projects-my-app/session_2026-06-05_09-14-32.md útvonalra íródik (a slug = az útvonal utolsó két szegmense - jellel összefűzve). A ~/.claude/session-maps/a1b2c3d4e5f6 alatti session map bejegyzés rögzíti ennek a fájlnak az útvonalát, 600 jogosultsággal.

2. példa — Prompt mentése

Trigger: egy aktív Claude Code sessionben promptot küldesz (automatikus az UserPromptSubmit hookon keresztül).

Tényleges bemenet (a hook által stdin-re küldött JSON payload):

{
  "session_id": "a1b2c3d4e5f6",
  "prompt": "Implement the next uncompleted task in plans/FEAT-001.md",
  "cwd": "/Users/alice/projects/my-app"
}

A session naplófájlhoz hozzáfűzött eredmény:

## 09:15:44

Implement the next uncompleted task in plans/FEAT-001.md

---

Minden további prompt egy új ## HH:MM:SS szakaszt fűz ugyanahhoz a fájlhoz, így a session promptjairól teljes, időrendi feljegyzés épül.
