Summary

Three shell scripts (prompt_log_lib.sh, prompt_log_new_session.sh, prompt_log_save.sh) that automatically capture every prompt submitted to Claude Code into per-project Markdown log files, organized by session.

When to use

These scripts run automatically — you do not invoke them manually during normal use. They are wired to Claude Code hooks in settings.json:

  • prompt_log_new_session.sh — called by the SessionStart hook whenever a new Claude Code session begins
  • prompt_log_save.sh — called by the UserPromptSubmit hook each time you submit a prompt
  • prompt_log_lib.sh — an internal shared library; sourced by the two scripts above, never invoked directly

Invoke them manually only when debugging the logging setup or when writing a script that needs to create or append to a session log file programmatically.

Invocation

prompt_log_new_session.sh — Session initialiser

Called automatically by the SessionStart hook. Reads a JSON payload from stdin.

echo '{"session_id":"abc123","cwd":"/path/to/project"}' | bash prompt_log_new_session.sh

Creates a new timestamped session log file under ~/.claude/prompt-logs/<project-slug>/session_YYYY-MM-DD_HH-MM-SS.md and writes a session map entry to ~/.claude/session-maps/<session_id>.

prompt_log_save.sh — Prompt appender

Called automatically by the UserPromptSubmit hook. Reads a JSON payload from stdin.

echo '{"session_id":"abc123","prompt":"your prompt text","cwd":"/path/to/project"}' | bash prompt_log_save.sh

Appends the prompt text with a HH:MM:SS timestamp heading to the session log file. If no session map exists yet, calls create_session_file internally (graceful late-init).

prompt_log_lib.sh — Shared library (internal)

Not invoked directly. Provides the create_session_file(session_id, cwd) function. Sourced at the top of both scripts above.

source "$HOME/.claude/scripts/prompt_log_lib.sh"

Example 1 — New session initialization

Trigger: A new Claude Code session starts in a project directory (automatic via SessionStart hook).

Effective input (JSON payload sent to stdin by the hook):

{
  "session_id": "a1b2c3d4e5f6",
  "cwd": "/Users/alice/projects/my-app"
}

Result:

# Prompts — 2026-06-05 09:14:32

**Session ID:** a1b2c3d4e5f6
**Resume:** `claude --resume a1b2c3d4e5f6`
**Session file:** `~/.claude/projects/Users-alice-projects-my-app/a1b2c3d4e5f6.jsonl`

---

The file is written to ~/.claude/prompt-logs/projects-my-app/session_2026-06-05_09-14-32.md (slug = last two path segments joined by -). The session map entry at ~/.claude/session-maps/a1b2c3d4e5f6 records the path to this file with permissions 600.

Example 2 — Prompt save

Trigger: You submit a prompt in an active Claude Code session (automatic via UserPromptSubmit hook).

Effective input (JSON payload sent to stdin by the hook):

{
  "session_id": "a1b2c3d4e5f6",
  "prompt": "Implement the next uncompleted task in plans/FEAT-001.md",
  "cwd": "/Users/alice/projects/my-app"
}

Result appended to the session log file:

## 09:15:44

Implement the next uncompleted task in plans/FEAT-001.md

---

Each subsequent prompt appends a new ## HH:MM:SS section to the same file, building a full chronological record of the session's prompts.
