scripts-plan

← Vissza az áttekintéshez

Összefoglaló

Shell scriptek csomagja, amely követi a terv előrehaladását, kikényszeríti az egy-feladat-egy-commit fegyelmet, auditálja a befejezett terv-futásokat, és (a Claude Code -cc változatban) SubagentStop hookon keresztül gátolja a subagent turn-end-jét commit nélkül — közvetlenül a fejlesztők, illetve automatikusan az /implement-next, /implement-all, /implement-next-cc és /implement-all-cc használja.

Mikor használd

Akkor használd ezeket a scripteket, amikor olyan Markdown terv-fájlokkal dolgozol, amelyek - [ ] / - [x] feladat-checkboxokat tartalmaznak:

  • Ha látni szeretnéd, melyik a következő feladat és mennyi az aktuális készültségi százalék — használd a plan-progress.sh scriptet.
  • Ha meg akarod számolni, hány feladat van még hátra egy futás indítása előtt — használd a count-uncompleted-tasks.sh scriptet.
  • Az /implement-next 7. lépésének self-verification ellenőrzésén belül — a check-task-commit.sh ellenőrzi, hogy legalább egy nem-merge commit készült.
  • Bármikor később, egy korábbi futás független auditálásához — használd az audit-plan-run.sh scriptet.
  • A -cc változat szkriptjeit (implement-next-stop-gate.sh, implement-next-state-write.sh, implement-next-state-clear.sh) az /implement-all-cc és a SubagentStop hook hívja — kézzel nem hívod őket.

Meghívás

Felhasználó által hívható scriptek

plan-progress.sh

Formázott haladás-fejlécet nyomtat a következő befejezetlen feladathoz. A progress-header-phased.template (fázisokra bontott tervek) vagy a progress-header-flat.template (lapos tervek) sablont használja.

bash ~/.claude/scripts/plan-progress.sh <plan_file>

Kilépési kódok: 0 = fejléc kinyomtatva + NEXT_TASK_* gépileg olvasható sorok; 1 = minden feladat kész; 2 = használati hiba vagy fájl nem található; 3 = nincs felismerhető feladat-szakasz fejléc.

count-uncompleted-tasks.sh

Megszámolja a felső szintű - [ ] sorokat a felismert feladat-szakaszon belül. Kiírja: number_of_uncompleted_tasks=<N>.

bash ~/.claude/scripts/count-uncompleted-tasks.sh <plan_file>

Kilépési kódok: 0 = siker; 2 = használati hiba vagy fájl nem található.

audit-plan-run.sh

Önálló, futás utáni audit. Ellenőrzi, hogy a <sha_start> óta keletkezett nem-merge commitok száma megegyezik-e a futásban befejezett feladatok számával (jelenlegi befejezett mínusz futás előtti befejezett).

bash ~/.claude/scripts/audit-plan-run.sh <plan_file> <sha_start>

Kilépési kódok: 0 = PASS; 1 = VIOLATION vagy eltérés; 2 = használati vagy git hiba.

verify-run-commits.sh (elavult)

Ellenőrzi, hogy pontosan <expected_count> darab nem-merge commit készült a <sha_start> óta. Könnyebb, mint az audit-plan-run.sh — nem diffeli a terv-fájlt. Elavult — az audit-plan-run.sh váltotta le; visszafelé kompatibilitás miatt marad a csomagban.

bash ~/.claude/scripts/verify-run-commits.sh <sha_start> <expected_count>

Kilépési kódok: 0 = PASS; 1 = VIOLATION; 2 = használati vagy git hiba.

check-task-commit.sh

Ellenőrzi, hogy legalább egy nem-merge commit készült, nem üres fájlváltozásokkal, a <sha_before> óta. Automatikusan hívja az /implement-next 7. lépése (portable) vagy az /implement-next-cc szülő-szintű ellenőrzései.

bash ~/.claude/scripts/check-task-commit.sh <sha_before>

Kilépési kódok: 0 = PASS; 1 = VIOLATION; 2 = használati vagy git hiba.

Claude Code -cc változat szkriptjei (hook + sentinel — felhasználó nem hívja)

implement-next-stop-gate.sh

Az /implement-all-cc által használt SubagentStop hook szkript. Olvassa a $CWD/.claude/implement-next-state.json sentinelt, az agentId-re szűr (így a beágyazott sub-sub-agentek átmennek), és nem engedi lezárni a külső /implement-next-cc subagent turnjét, amíg új commit nem létezik a START_SHA óta. A ~/.claude/settings.json hooks.SubagentStop alatt regisztrálva.

implement-next-state-write.sh

Sentinel-író, amelyet az /implement-all-cc 4. lépése (gate felfegyverzése) hív. Kiír egy $CWD/.claude/implement-next-state.json fájlt a start SHA-val, terv-úttal, következő task névvel és agent ID-vel, hogy a hook olvashassa.

bash ~/.claude/scripts/implement-next-state-write.sh <cwd> <start_sha> <plan_path> <task_name> <agent_id>

implement-next-state-clear.sh

Sentinel-tisztító, amelyet az /implement-all-cc 6. lépése (mindig, a recovery check után) hív. Törli a $CWD/.claude/implement-next-state.json fájlt, hogy a hook ne gátolja idő előtt a következő iteráció subagentjét.

bash ~/.claude/scripts/implement-next-state-clear.sh <cwd>

Belső függőségek (a felhasználó által hívható scriptek hívják — közvetlenül nem indítandók)

  • task_section.awk — AWK töredék, amely akkor aktiválódik, amikor a parser belép egy felismert feladat-szakasz fejlécbe (## Tasks, ## Task vagy ## Task breakdown). Hívja a plan-progress.sh, a count-uncompleted-tasks.sh és az audit-plan-run.sh.
  • progress-header-phased.template — Mustache-stílusú sablon fázisokra bontott tervekhez (azaz olyanokhoz, ahol vannak ### fázis-fejlécek). Hívja a plan-progress.sh.
  • progress-header-flat.template — Mustache-stílusú sablon lapos tervekhez (nincsenek fázis-fejlécek). Hívja a plan-progress.sh.

Függőségi táblázat

Script Típus Belső függőségek Automatikusan hívja
plan-progress.sh user task_section.awk, sablonok /implement-next 1. lépés
count-uncompleted-tasks.sh user task_section.awk — (kézi pre-run sanity check)
audit-plan-run.sh user task_section.awk —
verify-run-commits.sh elavult — — (az audit-plan-run.sh váltotta le; backwards compat miatt)
check-task-commit.sh user — /implement-next 7. lépés (portable) vagy /implement-next-cc szülő-szintű ellenőrzések
implement-next-stop-gate.sh hook (cc) sentinel json SubagentStop hook a ~/.claude/settings.json-ban (gátolja az /implement-all-cc-t)
implement-next-state-write.sh internal (cc) — /implement-all-cc 4. lépés (gate felfegyverzése)
implement-next-state-clear.sh internal (cc) — /implement-all-cc 6. lépés (mindig, recovery check után)
task_section.awk internal — plan-progress, count-uncompleted, audit-plan-run
progress-header-phased.template internal — plan-progress.sh
progress-header-flat.template internal — plan-progress.sh

1. példa — Terv-előrehaladás ellenőrzése a plan-progress.sh scripttel

Bemenet:

bash ~/.claude/scripts/plan-progress.sh plans/FEAT-001-skill-command-script-handouts.md

Kimenet / Eredmény:

Progress   : [█████████░░░] 79%  (15/19 tasks, 3/6 phases)
Next task  : 4.1  **File**: `handout/scripts-plan.html`

NEXT_TASK_NAME=**File**: `handout/scripts-plan.html`
NEXT_TASK_PHASE=4
NEXT_TASK_NUMBER=1

Kilépési kód: 0. A NEXT_TASK_* sorok gépileg olvashatók, és az /implement-next az 1. lépésben ezeket parsolja, hogy azonosítsa a megvalósítandó feladatot.

2. példa — Befejezett futás auditálása az audit-plan-run.sh scripttel

Bemenet:

bash ~/.claude/scripts/audit-plan-run.sh plans/FEAT-001-skill-command-script-handouts.md abc1234

Kimenet / Eredmény (PASS):

Plan file : plans/FEAT-001-skill-command-script-handouts.md
sha_start : abc1234
Tasks completed in this run (top-level) : 4
Non-merge commits since sha_start : 4

PASS: 4 commits for 4 completed tasks — counts match

Kimenet / Eredmény (VIOLATION):

Plan file : plans/FEAT-001-skill-command-script-handouts.md
sha_start : abc1234
Tasks completed in this run (top-level) : 4
Non-merge commits since sha_start : 3

VIOLATION: 3 commits for 4 completed tasks — counts do not match

A VIOLATION azt jelenti, hogy legalább két feladat egyetlen commitba lett összevonva, ami megsérti az egy-feladat-egy-commit szabályt. Kilépési kód: 1.

Kapcsolódó eszközök

  • /implement-next (portable) — Az 1. lépésben meghívja a plan-progress.sh-t, a 7. lépés self-verification részében pedig a check-task-commit.sh-t.
  • /implement-next-cc (Claude Code) — Ugyanaz a hívásmód; ráadásul a SubagentStop hook (implement-next-stop-gate.sh) gátolja, amikor az /implement-all-cc-ből hívják.
  • /implement-all (portable) — Iterációnként hívja a plan-progress.sh-t és záró auditként az audit-plan-run.sh-t.
  • /implement-all-cc (Claude Code) — Ugyanazok a hívók, plusz a sentinel-pár implement-next-state-write.sh / implement-next-state-clear.sh minden subagent indítás körül.
