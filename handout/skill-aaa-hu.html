Összefoglaló

Ötletek, termékfunkciók, architektúra és kód értékelése és továbbfejlesztése bizonyítékok, iparági legjobb gyakorlatok, kutatás, hézagelemzés és 3–4 világszínvonalú opció segítségével.

Mikor használd

Akkor használd az /aaa parancsot, ha tudni szeretnéd, hogy egy ötlet, funkció, architektúra, munkafolyamat, stratégia vagy kódrészlet kiváló-e — vagy hogyan emelhető a lehető legmagasabb gyakorlati minőségi szintre. Szigorú termék-, UX-, technikai, architektúra- vagy kódbírálatot ad, amelyet kutatás és közvetlen ítéletalkotás támaszt alá.

Triggerkifejezések (a SKILL.md-ből):

  • "is this excellent?"
  • "is this world-class?"
  • "AAA quality"
  • "worth building?"
  • "how do I improve this?"
  • "rigorous review of this feature / architecture / code"

Meghívás

Hívd meg közvetlenül slash parancsként, opcionálisan az értékelendő dolog leírását követve:

/aaa [description or paste of the idea / feature / code / architecture to evaluate]

A skill bármilyen artefaktumot elfogad: egy rövid ötletet, egy funkcióspecifikációt, egy kódrészletet, egy architektúradiagram leírását vagy egy teljes fájlt. Ha további kontextus (követelmények, stack, versenytársak) elérhető, vedd bele — a skill fel fogja használni.

1. példa — Ötletminőség értékelése

Bemenet:

/aaa We want to build an AI-powered note-taking app that auto-links related notes and surfaces forgotten ideas at the right moment. Is this AAA quality?

Kimenet / Eredmény:

Initial classification: Promising but underdeveloped

The core idea (auto-linking + surfacing) is validated by tools like Roam, Obsidian, and Mem.
Differentiation is low without a clear edge on timing or context.

Gap analysis:
- Problem intensity: high (note retrieval is a known pain)
- Differentiation: weak — Mem.ai already does this with LLMs
- Feasibility: medium (embedding-based retrieval is mature)

3 world-class options:
1. Lean/elegant — Obsidian plugin, no new app, ship fast
2. AI-native — real-time bidirectional linking with semantic drift detection
3. Enterprise — team knowledge graph with access controls

Recommendation: Option 1 to validate, Option 2 if differentiation from Mem is confirmed.

2. példa — Kódbírálat

Bemenet:

/aaa
```python
def get_user(id):
    result = db.execute(f"SELECT * FROM users WHERE id = {id}")
    return result.fetchone()
```
Is this world-class code?

Kimenet / Eredmény:

Classification: Weak

Blockers:
- SQL injection: f-string interpolation directly into a query is a critical security vulnerability.
  Fix: use parameterized queries — db.execute("SELECT * FROM users WHERE id = ?", (id,))
- No error handling: db failures silently return None

Improvements:
- Type hints: def get_user(id: int) -> Optional[User]
- Limit columns: SELECT id, name, email instead of SELECT *
- Use an ORM if available (SQLAlchemy, Django ORM)

Recommendation: Fix the injection blocker immediately; add type hints and column specificity next.

Kapcsolódó eszközök

  • /da-review — Egymenetes ördög ügyvédje bírálat, amely megtalálja a hibákat anélkül, hogy javításokat alkalmazna; jó egy gyors kritikus átolvasáshoz az /aaa előtt.
  • /iterative-review — Több ügynökös bírálati hurok, amely a közepes vagy súlyosabb problémákat egyszerre megtalálja és javítja; akkor használd, ha azt szeretnéd, hogy a problémák automatikusan megoldódjanak, ne csak felsorolva legyenek.
