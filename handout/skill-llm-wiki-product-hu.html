/llm-wiki-product

← Vissza az áttekintéshez

Összefoglaló

Állíts be vagy üzemeltess egy termékcsapatra szabott LLM Wikit (Karpathy-mintára) — egy tartós, LLM által karbantartott markdown tudásbázist szoftverprojekt-tartalmakhoz: funkciókhoz, specifikációkhoz, ütemtervekhez, architekturális döntésekhez, kódjegyzetekhez és versenytárs-dokumentumokhoz.

Megjegyzés: nem termékorientált területekhez (kutatás, személyes tudás, könyvolvasás, kurzusjegyzetek) használd helyette a /llm-wiki skillt. A két skill egy projekten belül kölcsönösen kizárja egymást.

Mikor használd

Akkor használd a /llm-wiki-product-ot, amikor olyan tudásbázist akarsz fenntartani, amely egy szoftveres terméket követ — annak funkcióit, specifikációit, döntéseit, kódjegyzeteit és versenytárs-elemzéseit. A Karpathy-féle wiki-mintát alkalmazza, csoportosított raw/ könyvtárakkal és wiki/features/, illetve wiki/decisions/ oldalkategóriákkal.

Trigger kifejezések (a SKILL.md alapján):

  • "set up an LLM wiki"
  • "create a product wiki"
  • "apply the Karpathy wiki pattern"
  • "ingest into the wiki"
  • "draft a spec from the wiki"
  • "create a feature page"
  • "lint the wiki"
  • Bármilyen feladat, amely az llm-wiki/ könyvtárat érinti raw/, wiki/, drafts/ rétegekkel, szoftverprojekt kontextusban
  • Tartós, LLM által karbantartott tudásbázis építése termékdokumentációkból, specifikációkból, forráskód-jegyzetekből és versenytárs-dokumentumokból

Mikor ezt használd a /llm-wiki helyett

  • /llm-wiki-product (ez a skill) — szoftverprojekt-wikik: funkciók, specifikációk, ütemtervek, döntések, versenytárs-dokumentumok. Csoportosított raw/-t (product-docs/, specs/, code-notes/, competitor-docs/) és wiki/features/, wiki/decisions/ oldalkategóriákat használ.
  • /llm-wiki — általános, bármilyen nem termékorientált terület: kutatás, személyes tudás, könyvolvasás, kurzusjegyzetek. Lapos raw/-t és wiki/topics/, wiki/decisions/ oldalkategóriákat használ.

Meghívás

Slash parancsként hívod meg. A skill automatikusan a Setup, Operate vagy Repair ágra irányít az llm-wiki/ állapota alapján:

/llm-wiki-product

A skill maga dönti el, melyik módot futtassa:

  • Nincs llm-wiki/ könyvtár → Setup fut (1. lépés): létrehozza a teljes könyvtárfát, bemásolja a sablonokat, a projekthez igazítja a schema.md-t (felderíti a meglévő dokumentációs gyökereket), és felajánlja a pointer beillesztését a CLAUDE.md/AGENTS.md fájlokba.
  • Az llm-wiki/ létezik és teljes (van schema.md-je és teljes rétegfája) → a kért Operation fut (2. lépés): ingest, funkciólap létrehozása, spec-piszkozat, lekérdezés, lint, archiválás, séma-evolúció vagy watcher parancsok.
  • Az llm-wiki/ létezik, de hiányos → Repair fut (1b lépés): pótolja a hiányzó részeket anélkül, hogy felülírná a meglévő fájlokat.

A Setup által létrehozott könyvtárszerkezet:

llm-wiki/
  karpathy-llm-wiki.md   ← reference copy
  schema.md              ← adapted to project
  index.md
  log.md
  raw/
    product-docs/
    specs/
    code-notes/
    competitor-docs/
  wiki/
    overview.md
    glossary.md
    features/
    decisions/
  drafts/
    specs/

A wiki beállítása után elérhető műveletek:

  • ingest — beolvas egy forrást, frissíti a wiki-oldalakat, frissíti az index.md-t, hozzáfűz a log.md-hez
  • create a feature page — szintetizál egy wiki/features/<name>.md oldalt a raw forrásokból
  • draft a spec — ír egy drafts/specs/<name>.md-t (elfogadásig nem számít termékigazságnak)
  • query — beolvassa az index.md-t, megkeresi az oldalakat, hivatkozásokkal szintetizálja a választ
  • lint — egészségellenőrzést végez a wikin (árvák, hiányzó hivatkozások, ellentmondások)
  • archive — elavult oldalt átmozgat a wiki/archive/ mappába, „superseded-by” jegyzettel
  • evolve schema — ha új oldalkategória jelenik meg, először a schema.md-t frissíti
  • start-watch / stop-watch / watch-status — fájlrendszer-watcher a raw/-hoz
  • check-pending — a függő és elhalasztott ingest-sor tételeinek áttekintése

2. példa — Funkciólap készítése egy elfogadott specifikációból

Szemléltető példa

Bemenet:

/llm-wiki-product
Create a feature page for "usage-based billing" from the spec in raw/specs/usage-billing-v2.md

Kimenet / Eredmény:

Running Operation: Create a feature page

Source: llm-wiki/raw/specs/usage-billing-v2.md
  Evidence level: Strong (accepted spec)

Created: llm-wiki/wiki/features/usage-based-billing.md
  Sections: Overview, Behaviour, Configuration, Edge Cases, Open Questions
  No conflicts detected with existing wiki content.

Updated: llm-wiki/index.md
  Added entry: "usage-based billing" → wiki/features/usage-based-billing.md

Appended to log.md:
  ## [2026-06-05] create | usage-based-billing.md | from raw/specs/usage-billing-v2.md

Next recommended operation: query or lint to verify wiki state.

Kapcsolódó eszközök

  • /llm-wiki — Ennek a skillnek az általános-területű változata, kutatáshoz, személyes tudáshoz, könyvolvasáshoz és kurzusjegyzetekhez. Egy projekten belül kölcsönösen kizárja egymást a /llm-wiki-product-tal — válaszd azt, amelyik a te területedhez illik.
