Summary

Set up or operate a product-team LLM Wiki (Karpathy pattern) — a persistent, LLM-maintained markdown knowledge base for software-project content: features, specs, roadmaps, architectural decisions, code notes, and competitor documents.

Note: For non-product domains (research, personal knowledge, book reading, course notes), use /llm-wiki instead. The two skills are mutually exclusive on a project.

When to use

Use /llm-wiki-product when you need to maintain a knowledge base that tracks a software product — its features, specs, decisions, code notes, and competitor intelligence. It applies the Karpathy wiki pattern with bucketed raw/ directories and page categories of wiki/features/ and wiki/decisions/.

Trigger phrases (from SKILL.md):

  • "set up an LLM wiki"
  • "create a product wiki"
  • "apply the Karpathy wiki pattern"
  • "ingest into the wiki"
  • "draft a spec from the wiki"
  • "create a feature page"
  • "lint the wiki"
  • Any task involving the llm-wiki/ directory with raw/, wiki/, drafts/ layers in a software project context
  • Building a persistent, LLM-maintained knowledge base from product docs, specs, source-code notes, and competitor documents

When to use this skill vs. /llm-wiki

  • /llm-wiki-product (this skill) — software-project wikis: features, specs, roadmaps, decisions, competitor docs. Uses bucketed raw/ (product-docs/, specs/, code-notes/, competitor-docs/) and page categories wiki/features/, wiki/decisions/.
  • /llm-wiki — generic, any non-product domain: research, personal knowledge, book reading, course notes. Uses flat raw/ and page categories wiki/topics/, wiki/decisions/.

Invocation

Invoke as a slash command. The skill routes to Setup, Operate, or Repair automatically based on the state of llm-wiki/:

/llm-wiki-product

The skill determines which mode to run:

  • No llm-wiki/ directory exists → runs Setup (Step 1): creates the full directory tree, copies templates, adapts schema.md to the project (detects existing docs roots), and offers pointer injection into CLAUDE.md/AGENTS.md.
  • llm-wiki/ exists and is complete (contains schema.md and full layer tree) → runs the requested Operation (Step 2): ingest, create feature page, draft spec, query, lint, archive, evolve schema, or watcher commands.
  • llm-wiki/ exists but is incomplete → runs Repair (Step 1b): fills in missing pieces without overwriting existing files.

Directory structure created by Setup:

llm-wiki/
  karpathy-llm-wiki.md   ← reference copy
  schema.md              ← adapted to project
  index.md
  log.md
  raw/
    product-docs/
    specs/
    code-notes/
    competitor-docs/
  wiki/
    overview.md
    glossary.md
    features/
    decisions/
  drafts/
    specs/

Operations available once the wiki is set up:

  • ingest — read a source, update wiki pages, update index.md, append to log.md
  • create a feature page — synthesize a wiki/features/<name>.md from raw sources
  • draft a spec — write a drafts/specs/<name>.md (not product truth until accepted)
  • query — read index.md, find pages, synthesize answer with citations
  • lint — health-check the wiki (orphans, missing citations, contradictions)
  • archive — retire a stale page to wiki/archive/ with a superseded-by note
  • evolve schema — when a new page category emerges, update schema.md first
  • start-watch / stop-watch / watch-status — filesystem watcher for raw/
  • check-pending — review pending and snoozed ingest queue entries

Example 1 — Setting up a product wiki for a SaaS project

Illustrative example

Input:

/llm-wiki-product
Set up a product wiki for our SaaS billing service.

Output / Result:

No llm-wiki/ directory found. Running Setup (Step 1).

Created:
  llm-wiki/
    karpathy-llm-wiki.md   ← reference copy
    schema.md              ← adapted to project (Documentation/ and adr/ detected)
    index.md
    log.md
    raw/
      product-docs/
      specs/
      code-notes/
      competitor-docs/
    wiki/
      overview.md

      features/
      decisions/

Detected external documentation roots:
  | Path             | Treat as           | Evidence level |
  |------------------|--------------------|----------------|
  | Documentation/   | raw/product-docs/  | Strong         |
  | adr/             | raw/product-docs/  | Strong         |

Log entry appended:
  ## [2026-06-05] init | wiki created | schema.md, index.md, wiki/overview.md, wiki/glossary.md

Offer: Inject llm-wiki pointer into CLAUDE.md? (yes/no)

Example 2 — Creating a feature page from an accepted spec

Illustrative example

Input:

/llm-wiki-product
Create a feature page for "usage-based billing" from the spec in raw/specs/usage-billing-v2.md

Output / Result:

Running Operation: Create a feature page

Source: llm-wiki/raw/specs/usage-billing-v2.md
  Evidence level: Strong (accepted spec)

Created: llm-wiki/wiki/features/usage-based-billing.md
  Sections: Overview, Behaviour, Configuration, Edge Cases, Open Questions
  No conflicts detected with existing wiki content.

Updated: llm-wiki/index.md
  Added entry: "usage-based billing" → wiki/features/usage-based-billing.md

Appended to log.md:
  ## [2026-06-05] create | usage-based-billing.md | from raw/specs/usage-billing-v2.md

Next recommended operation: query or lint to verify wiki state.
